# Iteration 02 - Bitrix24 Read Sync

## Goal

Реализовать загрузку лидов, контактов, стадий и активностей из Bitrix24 в нашу систему.

## Scope

- Добавить env-переменные для Bitrix24.
- Реализовать Bitrix REST client.
- Добавить ручной sync endpoint для администратора.
- Сохранять raw payload входящих объектов.
- Делать upsert по внешним id Bitrix24.
- Логировать каждый sync в `IntegrationEvent`.
- Добавить базовую обработку ошибок и лимитов.

## Out of scope

- Не делать запись обратно в Bitrix24.
- Не делать Telegram-сценарии.
- Не строить сложный scheduler.
- Не делать полную синхронизацию всех сущностей Bitrix24, если они не нужны для MVP.

## Definition of Done

- Администратор может запустить sync вручную.
- Повторный sync не создает дубли.
- Лиды обновляются при изменении данных.
- Ошибки Bitrix24 сохраняются в журнал интеграции.
- Сырые payload доступны для диагностики.

## Verification

```bash
cd backend
npm run build
```

Ручная проверка:

- запустить sync endpoint;
- проверить количество лидов до и после повторного sync;
- изменить тестовый лид в Bitrix24 и убедиться, что данные обновились;
- проверить записи `IntegrationEvent`.
