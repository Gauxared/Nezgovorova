# Iteration 05 - Bitrix24 Write-back

## Goal

Передавать результаты работы из нашей системы обратно в Bitrix24 контролируемым и журналируемым способом.

## Scope

- Добавить outbound command модель для записи в Bitrix24.
- Реализовать добавление комментария в таймлайн лида.
- Реализовать создание или закрытие CRM-активности.
- Реализовать ограниченное изменение стадии лида по разрешенным переходам.
- Добавить retry для временных ошибок API.
- Логировать результат каждой записи в `IntegrationEvent` и `AuditLog`.

## Out of scope

- Не автоматизировать сложные бизнес-процессы Bitrix24.
- Не менять стадии без явного правила.
- Не удалять данные в Bitrix24.

## Definition of Done

- Вечерний комментарий сотрудника может быть записан в Bitrix24.
- Закрытие задачи в Telegram может закрыть CRM-активность.
- Ошибка записи не теряет outbound command.
- Повтор команды не создает дубли в Bitrix24.
- Руководитель видит статус записи обратно.

## Verification

```bash
cd backend
npm run build
```

Ручная проверка:

- отправить отчет по тестовому лиду;
- инициировать запись комментария в Bitrix24;
- проверить таймлайн лида в Bitrix24;
- повторить команду и проверить отсутствие дубля;
- проверить журнал интеграции.
