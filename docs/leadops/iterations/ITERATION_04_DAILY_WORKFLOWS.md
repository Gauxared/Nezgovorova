# Iteration 04 - Daily Workflows

## Goal

Реализовать ежедневную работу сотрудника через Telegram: утренний план и вечерний отчет.

## Scope

- Добавить правила расписания для утренних и вечерних сообщений.
- Сформировать выборку лидов и активностей сотрудника на сегодня.
- Отправлять утреннее сообщение со списком задач.
- Реализовать кнопки быстрых ответов.
- Реализовать вечерний опрос по выполнению задач.
- Сохранять ответы в `WorkerDailyReport` и `WorkerReportItem`.
- Показывать отчеты в admin/manager интерфейсе или read-only API.

## Out of scope

- Не писать результаты обратно в Bitrix24.
- Не делать сложный NLP для свободного текста.
- Не делать многоязычность.

## Definition of Done

- Сотрудник получает утренний список задач.
- Сотрудник получает вечерний опрос.
- Ответы сохраняются по каждому лиду или активности.
- Руководитель видит, кто не отправил отчет.
- Повторная отправка сообщения не создает дубли отчетов.

## Verification

```bash
cd backend
npm run build
```

Ручная проверка:

- назначить сотруднику несколько лидов;
- запустить ручной endpoint утренней рассылки;
- ответить в Telegram;
- запустить вечерний опрос;
- проверить сохраненные отчеты в БД.
