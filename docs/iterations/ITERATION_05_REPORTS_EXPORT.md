# Iteration 05 — Reports and Export

## Goal

Реализовать модуль отчетов: создание, просмотр, фильтрация и экспорт.

## Scope

### Report types

- Отчет по проектам;
- Отчет по задачам;
- Финансовый отчет;
- Отчет по клиентам;
- Сводный управленческий отчет.

### Backend endpoints

- GET `/api/reports`;
- POST `/api/reports`;
- GET `/api/reports/:id`;
- PATCH `/api/reports/:id`;
- DELETE `/api/reports/:id`;
- GET `/api/reports/:id/export?format=csv`;
- GET `/api/reports/:id/export?format=xlsx`, если не усложнит проект.

### Report behavior

Отчет хранит:

- title;
- description;
- type;
- filters;
- createdById.

Важно:

- отчет хранит настройки, а не статичный результат;
- данные пересчитываются при открытии;
- экспорт строится на актуальных данных.

### Frontend

- `/reports`;
- `/reports/:id`;
- список отчетов;
- форма создания;
- выбор типа;
- фильтры;
- просмотр отчета;
- таблица результатов;
- график, если уместно;
- кнопка экспорта.

## Out of scope

Не делать:

- drag-and-drop конструктор;
- пользовательские формулы;
- расписание отправки отчетов;
- PDF-экспорт, если усложняет.

## Definition of Done

- можно создать отчет;
- можно выбрать тип;
- можно задать фильтры;
- отчет сохраняется в БД;
- отчет открывается по `/reports/:id`;
- данные пересчитываются при открытии;
- CSV экспорт работает;
- экспорт пишется в audit log;
- frontend показывает loading/error/empty;
- доступ ограничен ролями.

## Verification commands

```bash
cd backend
npm run dev
```

```bash
cd frontend
npm run dev
```

## Manual check

1. открыть `/reports`;
2. создать финансовый отчет;
3. указать период;
4. открыть отчет;
5. экспортировать CSV;
6. проверить audit log.

## Notes for Codex

- XLSX добавлять только если CSV уже стабилен.

## Общий формат ответа Codex после реализации

Codex должен указать:

- что сделано;
- какие файлы созданы или изменены;
- какие команды проверки выполнены;
- какие пункты Definition of Done выполнены;
- какие ограничения или проблемы остались.
