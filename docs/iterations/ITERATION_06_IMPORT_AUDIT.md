# Iteration 06 — Import and Audit Log

## Goal

Добавить импорт CSV и полноценный просмотр журнала действий.

## Scope

### Import

CSV-импорт для:

- clients;
- projects;
- tasks;
- financial_values.

Endpoints:

- POST `/api/import/clients`;
- POST `/api/import/projects`;
- POST `/api/import/tasks`;
- POST `/api/import/financial-values`.

Требования:

- проверять структуру файла;
- валидировать строки;
- показывать ошибки по строкам;
- не импортировать некорректные строки без сообщения;
- писать результат импорта в AuditLog или ImportLog;
- описать ожидаемые колонки.

### Audit log

Endpoint:

- GET `/api/audit-logs`.

Frontend:

- `/admin/audit`;
- таблица журнала;
- фильтры по пользователю, действию, сущности, периоду;
- просмотр старого и нового значения, если применимо.

Actions:

- create;
- update;
- delete;
- login;
- import;
- export.

## Out of scope

Не делать:

- Excel-импорт, если CSV работает;
- UI сопоставления колонок;
- откат импорта;
- сложный diff-viewer.

## Definition of Done

- можно импортировать клиентов;
- можно импортировать проекты;
- можно импортировать задачи;
- можно импортировать финансовые записи;
- ошибки импорта понятны;
- успешный импорт пишется в журнал;
- экспорт отчета пишется в журнал;
- login пишется в журнал;
- `/admin/audit` показывает действия;
- фильтры журнала работают;
- доступ к журналу только у admin и analyst.

## Verification commands

```bash
cd backend
npm run dev
```

```bash
cd frontend
npm run dev
```

## Manual check

1. импортировать корректный CSV;
2. импортировать CSV с ошибками;
3. проверить сообщения;
4. открыть `/admin/audit`;
5. проверить запись импорта.
