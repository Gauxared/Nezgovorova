# Iteration 07 — Final Polish and Defense Readiness

## Goal

Довести проект до состояния, пригодного для демонстрации на защите.

Это итерация стабилизации, документации и UI-полировки, а не добавления крупных новых модулей.

## Scope

### Stability

Проверить:

- Docker/PostgreSQL;
- миграции;
- seed;
- backend;
- frontend;
- вход под тестовыми пользователями;
- CRUD;
- dashboard;
- отчеты;
- экспорт;
- импорт;
- audit log.

### UI polish

Улучшить:

- отступы;
- кнопки;
- таблицы;
- формы;
- empty states;
- error states;
- loading states;
- sidebar;
- header;
- страницы 404 и access denied.

### Documentation

Обновить README:

- описание проекта;
- стек;
- структура;
- запуск Docker/backend/frontend;
- миграции;
- seed;
- тестовые пользователи;
- роли;
- основные возможности;
- demo-сценарий.

### Seed

Данных должно хватать для демонстрации:

- 5–10 клиентов;
- 8–15 проектов;
- 30–50 задач;
- финансы за несколько периодов;
- разные статусы;
- просрочки;
- прибыльные и убыточные проекты;
- пользователи разных ролей.

## Out of scope

Не делать:

- переписывание архитектуры;
- новые крупные модули;
- смену стека;
- сложные BI-конструкторы;
- лишние декоративные эффекты.

## Definition of Done

- проект запускается по README;
- seed создает demo-данные;
- admin-вход работает;
- роли проверены;
- CRUD работает;
- dashboard показывает реальные данные;
- отчеты создаются и открываются;
- экспорт работает;
- импорт работает;
- audit log работает;
- UI выглядит целостно;
- README актуален;
- demo-сценарий готов;
- Codex перечислил оставшиеся ограничения.

## Verification commands

```bash
docker compose up -d
cd backend
npm install
npx prisma migrate dev
npm run seed
npm run dev
```

```bash
cd frontend
npm install
npm run dev
npm run build
```

Codex должен указать:

- что сделано;
- какие файлы созданы или изменены;
- какие команды проверки выполнены;
- какие пункты Definition of Done выполнены;
