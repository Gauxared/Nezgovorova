# Architecture Decisions

## ADR-001 — Fullstack monorepo

Проект хранится в одном репозитории с папками `/frontend` и `/backend`.

## ADR-002 — PostgreSQL + Prisma

Данные имеют реляционную структуру: клиенты, проекты, задачи, пользователи, финансы, справочники и отчеты.

## ADR-003 — REST API

REST проще документировать, тестировать и объяснять на защите.

## ADR-004 — Расчеты аналитики на backend

Frontend не рассчитывает итоговые показатели, а только отображает данные.

## ADR-005 — Ролевая модель доступа

Права пользователей проверяются на backend.

## ADR-006 — Seed-данные обязательны

Система должна быть готова к демонстрации сразу после запуска.

## ADR-007 — Audit log

Создание, изменение, удаление, импорт, экспорт и вход в систему записываются в журнал.

## ADR-008 — Отчеты хранят настройки

Отчет хранит тип и фильтры, а данные пересчитываются при открытии.

## ADR-009 — Backend на Express

Для backend выбран Express + TypeScript. Express соответствует утвержденному стеку, проще для стартового каркаса, REST API и демонстрации на защите. NestJS пока не используется, чтобы не усложнять первую итерацию лишней инфраструктурой.
