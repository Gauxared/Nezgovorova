# Demo Scenario for Defense

## Подготовка

1. Запустить PostgreSQL через Docker.
2. Применить миграции.
3. Выполнить seed.
4. Запустить backend.
5. Запустить frontend.
6. Войти под администратором:

```text
admin@example.com
admin12345
```

## Сценарий показа

1. Вход в систему под `admin@example.com / admin12345`.
2. Dashboard: KPI, графики, просроченные задачи, проблемные проекты.
3. Клиенты: список, поиск, создание клиента.
4. Проекты: создание или редактирование проекта, выбор клиента/статуса/направления.
5. Задачи: создание задачи, выбор проекта и ответственного.
6. Финансы: добавление дохода или расхода; шаг ввода сумм — 10 000 рублей.
7. Пересчет аналитики на dashboard.
8. Отчеты: создание финансового отчета с фильтрами.
9. Открытие отчета и экспорт CSV.
10. Импорт CSV клиента из `docs/import_samples/clients.csv`.
11. Журнал действий: показать login, create, export, import.
12. Проверка роли с ограниченным доступом, например `finance@example.com / finance12345`.

## Что подчеркнуть на защите

- единая база данных;
- справочники и унификация;
- расчет показателей на backend;
- визуализация;
- отчеты и экспорт;
- роли и защита;
- audit log;
- возможность дальнейшего расширения.

## Быстрая Проверка Готовности

- [ ] Backend health возвращает `database: connected`.
- [ ] Admin-вход работает.
- [ ] Dashboard загружает реальные данные.
- [ ] CRUD сохраняет записи после обновления страницы.
- [ ] Отчет создается и открывается.
- [ ] CSV экспорт скачивается.
- [ ] CSV импорт показывает результат и ошибки по строкам.
- [ ] Audit log содержит действия пользователя.
