# Codex Workflow

## Запрос для режима планирования

```text
Прочитай файл /docs/iterations/ITERATION_XX_NAME.md.

Работай в режиме планирования.

Сначала НЕ пиши код. Сначала:
1. кратко перескажи цель итерации;
2. проверь, все ли требования понятны;
3. предложи план реализации по шагам;
4. перечисли файлы и папки, которые будут созданы или изменены;
5. укажи риски и спорные места;
6. сопоставь каждый пункт Definition of Done с конкретными действиями.

После этого остановись и дождись моего подтверждения.
```

## Запрос на реализацию

```text
План принимаю. Реализуй итерацию строго по файлу /docs/iterations/ITERATION_XX_NAME.md.

Важно:
- не выходи за Scope;
- не делай пункты из Out of scope;
- не переписывай проект без необходимости;
- обнови документацию, если изменились команды запуска или структура.

После реализации:
1. проверь Definition of Done;
2. выполни доступные команды проверки;
3. кратко напиши, что сделано;
4. укажи, что не сделано;
5. напиши, как проверить результат.
```

## Запрос на исправление

```text
При проверке возникла ошибка:

[вставить ошибку]

Исправь только причину этой ошибки. Не меняй архитектуру и не добавляй новую функциональность.
```

## Запрос на ревью

```text
Проведи ревью текущей итерации. Проверь Definition of Done, выход за Scope, невыполненные требования, ошибки запуска и риски перед следующей итерацией. Не пиши новую функциональность.
```
